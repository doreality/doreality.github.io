Lecture 01: Overview

Reviewed: Yes
Week: Week 01

课本和前言

课本：

  • from 1998 213课程 as course note
  • 版本：第三版
  • 和课程内容 tied together

课程目的：代码到底怎么运行的——在 box 里面，box 在做什么？

  1. 工具、技巧、方法——be better
    1. 程序如何运行
    2. 机器如何支持程序的运行
    3. 为什么程序有时能正常运行有时却不能
  2. 为后续的课程作准备

Great Reality #1: 数字在计算机中的表示

linux： gdb

mac：lldb

整数的溢出（overflow）和浮点数的舍入（roundoff）：

use finite representations of things that are potentially infinite in their expanse.

32767 + 1 == -32768

corner cases: 临界情况

安全泄漏 / 重要的程序，在任何遇到临界的情况，都要好好了解。

Great Reality #2: 汇编和目标程序（指令系统）

机器级编程

汇编语言

获取 C 编译器所编译的汇编代码，阅读并理解（而不是自己写汇编语言程序）

Intel processor

x86-64 64位处理器

Great Reality #3: 分层存储系统

C 不做 bounds checking（边界检查）

This can be some of the worst debugging nightmares that exists on earth is to try and figure out memory referencing errors.

  1. 不使用 C 或 C++ 编程
  2. bullet proof “防弹测试”

理解数据结构的机器级别的表示，以及它们如何运行。

Great Reality #4: 更加关注程序的性能

asymptotic complexity 渐进复杂度

对于程序，一方面要使用合适的数据结构和算法，另一方面，也要关注底层运行时，到底发生了什么，如何从编程角度提高性能。

复制矩阵，行优先和列优先。右边的运行时间大概是左边的20倍。

封面：四种不同的内存访问模式（后续会更深入讨论）

复制程序的吞吐量以 MB/s 位单位的。

row by row much better ➡️ 内存层次中的缓存结构

Great Reality #5: 互联的计算机

不仅仅是执行程序，而且可以和其他计算机互联，交流。

interact with physical world

System 的课程：

  1. Builder-Centric：怎么去设计，实现一个处理器/操作系统/……
  2. Programmer Perspective：
    1. 作为在机器上写代码的人，更了解所写的代码运行的环境；
    2. 实现功能时，了解机器的特性，以及如何很好地依据计算机的特性来实现
    3. get effective tools
